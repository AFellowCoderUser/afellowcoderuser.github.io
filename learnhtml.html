Learning HTML

HTML is the basic skeleton of a website. HTML is a text document that tells whatever browser you are using to display the information written on the file. A website simply cannot exist without HTML files because there is no file to be displayed in the browser.

This guide will take you through the basic steps of HTML.

Getting Started on HTML

You may create another HTML file on NeoCities, or in general just use any HTML file. I recommend creating a new blank HTML file.

With your new HTML file created, your new HTML file on NeoCities should have some code already pre-written. If your new HTML file isn't on NeoCities, here's what should be on your HTML file:

<!DOCTYPE html>
<html>
    <head>
        <meta charset="UTF-8">
        <title>Your Page Title</title>
        <link rel="stylesheet" type="text/css" href="/style.css" />
        <meta name="viewport" content="width=device-width, initial-scale=1.0">
    </head>
    <body>
        <p>This is a paragraph</p>
    </body>
</html>

Copy and paste the code into your HTML file. This is the base of a website page. Now, let's walk through what all of that code means.

    <!DOCTYPE html> tells the browser that this is a HTML file. While webpages can work without this declaration, without it, it'll cause some weird behavior.

    <html> is used to contain the <head> and <body>. Sometimes, it may also say <html lang="en">, this states that the page is in english.

    <head> is what contains meta information of your site. <meta charset="UTF-8"> is used to specify that the webpage uses the UTF-8 character set. This is needed to display symbols. The second meta tag is used to make sure that the webpage (sometimes) looks good on mobile. <title>Your Page Title</title> is the page's title displayed on the tab. Finally, the <link> is used to link CSS files to the page, it's also used to link images to become the website's favicon.

    <body> is where all of the webpage's content is held in. Most of the website's code will be placed there.

    Finally, it is closed off with </html>. Nothing should be outside either the <head> or <body> tags.

This is the base of a webpage. The body is where you'll start and build upwards, slowly progressing until it reaches to the part where you like it.

Linking Files

Linking a HTML file to a CSS/JS/Whatever file is similar to how file directories work on your computer.

There are two ways to link a file to another file: absolute and relative.

Whenever you visit a website, the slash (/) after the domain is now the file path to the HTML file you are viewing on your browser.

Absolute Links

Absolute links are full links that contain the root of where the file is coming from. For example:

C:\file_name

or

https://yourwebsite.com/index.html

On the first example, C:\ is the source on where "file_name" is coming from. On the second example, https://yourwebsite.com is the source on where "index.html" is coming from.

You can use absolute links to link files to other files. However, you must use the absolute link when you are linking a file that is outside of your website's storage. This is called hotlinking, and it is usually disliked by many other websites, unless it is a website dedicated to file hosting.

Relative Links

Relative links are links that are relative to the page you're working on. For example:

\file_name

or

/index.html

Relative links are very similar to absolute links, the only difference is that it is missing the root of where the file is coming from. Relative links are used only when a file is linking a file that is within the same storage.

On NeoCities, whenever you are linking a file, it starts from the home folder first.

IDs and Classes

IDs and Classes are used to assign a specific ID or Class to a html tag. These are usually used by external CSS files to give that tag a style that is specified within the CSS file. Now, what is the difference to IDs and Classes?

    The id="" can be used multiple times, but it must have a unique ID. There cannot be two or more IDs that are the same.

    The class="" can be used multiple times in a HTML file, whether if it's the same or not.

We'll talk more about IDs and Classes on the CSS page, in fact, you're done reading this page! Now, let's learn how to give your website style.

HTML Tips

A HTML page can have a second <html> tag. People would utilize this second <html> tag to make a sidebar that is entirely separate from the main page. It's up to you to decide what the second <html> tag should be its purpose.

A HTML file can entirely work on its own without the use of an external CSS or JavaScript file. Not just the page being entirely just HTML, but you can include the <style> or <script> within the <head>. This can be useful if a single HTML file is only going to be using that CSS or JavaScript code. A tag can also have it own style (ex. <p style="text-align:center;">). We'll talk more about this on the CSS page.

Use <marquee> to make an infinitely scrolling text. Although deprecated, it still may work on different browsers.

To make a link that sends you to a section of a page, give a tag an id and name it to whatever you like (e.g. <p id="section">). In your page navigation bar, add <a href="#section">Section 1<a>. The hashtag (#) is followed by the id's name that you specified earlier.

To make a comment on your HTML file. Use <!--Your Text Here-->.
