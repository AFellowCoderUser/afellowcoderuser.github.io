Very Basics to CSS

Here, I'll be explaining about the CSS syntax and other handy things.

A CSS ruleset will look like something like this:

h1 {
    font-weight: bold
}

Here, this ruleset here says that all <h1> tags will be in bold without the use of the <b> tag. You shouldn't have any empty rulesets present in your CSS file. The last shorthand property (In the example code it's font-weight) may have no semicolon on the end. However, if you want to add more properties to the ruleset, first add a semicolon to the previous property and repeat on the others.

Like this:

h1 {
    font-weight: bold; /*Semicolon added because there is another property added below.*/
    color: white; /*Semicolon added. Is optional until another property is added.*/
}

Inheritance

When you apply a style to a HTML tag, the child tags will automatically inherit the parent tag's styles. However, this applies to every child tag. For example:

p {
    color: #00ffc8;
}

#box {
    color: #00b7ff;
    border: 2px solid #00b7ff;
}

#box p {
    color: inherit;
}

The code says that all paragraph text will be colored in a turquoise color, including those in the box. You can make the paragraph text inherit the color specified in the box with inherit.

However, this only applies to <p> tags. The reason why is because the p ruleset says that all paragraph text will be colored turquoise rather than all text tags. You can eliminate the use of inherit by moving color: #00ffc8; into the body ruleset. Now all text outside of the box will be turquoise, but all text inside the box will be light blue instead.

:root and *

The :root selects the root element of a HTML file. It is very similar to html and works like it. The only difference is that :root is a pseudo-class selector.

The * is the universal selector. Any property that you put under the universal selector will apply that property on every HTML tag if possible. I recommend using * to contain the property box-sizing: border-box;.

Styles to Specific Tags

You can assign a specific style to either a HTML tag, ID, or Class. For example:

p {
    color: red;
}

This code says that all <p> tags with text between the tags will be colored red.

#red {
    color: red;
}

This code says that all elements that are given the id="red" attribute will color the text within it red. IDs that have the same value are not to used more than once.

.red {
    color: red;
}

This code says that all elements that are given the class="red" attribute will color the text within it red. Classes with the same value can be used more than once.

CSS without CSS files

HTML should already recognize CSS code, whether it's coming from an external source or within the HTML file. You can either use the <style></style> tags in the <head>, or use style="" within a HTML tag. For example:

<style>
    .center {
        text-align: center;
    }
</style>

This is a very similar way to how you would write CSS code in an actual CSS file, except this CSS code is only present in the HTML file you are currently editing right now. This is useful if you are planning for the CSS code to only be used in one HTML file. If you want a specific tag to have this style, without the use of IDs or Classes, you can use:

<p style="text-align:center;"></p>

Here, only that specific <p> tag will have its text aligned to the center. I recommend using this method of styling if you're planning not to use this multiple times, especially when it's long and if you are planning to change the style. (That's when Classes come in.)

CSS Tips

You can make a page responsive with @media only screen and (max or min-width: 750px). This code will apply and override rulesets that you have listed within when the screen's width is below or above 700px. Example here, scroll to the bottom. Note that this can only work when using display:flex; or display:grid;.

You can customize the scrollbar with ::webkit-scrollbar. Note that this is not supported on all browsers.

Use only display:flex; or display:grid; when you are making a responsive website. To make a unresponive website (not recommended, unless for aesthetic purposes) use tables. Working with tables is a bit harder.
